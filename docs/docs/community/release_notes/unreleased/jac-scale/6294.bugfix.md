- **Fix: Archetypes with empty-dict defaults now persist correctly on MongoDB 6.x**: Nodes and edges with `has field: dict = {}` silently failed to update after the first write on MongoDB 6.x (the version jac-scale provisions on Kubernetes). When the field was dirty, `_put_node_atomic` routed it through the aggregation-pipeline `$set`, which rejects empty embedded documents with error 40180; the exception was swallowed and the write was lost. The dirty archetype fields are now stripped of empty-dict leaves before the pipeline `$set`, matching the existing `non_arch_top` handling. (MongoDB 7.0+ removed the 40180 restriction, so newer deployments were unaffected.)
