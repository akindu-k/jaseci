- **Fix: Eliminate intermittent `GET /` 500 failure under parallel test runs**: `JacAPIServer` now snapshots its routing config from its own `base_path` at startup (resolving profile and `jac.local.toml` overlays the same way the CLI does) instead of reading the shared process-level singleton on each request. This prevents `set_config()` mutations in one in-process client from affecting unrelated servers in the same process, while still honoring per-project overlay config in the request router.
