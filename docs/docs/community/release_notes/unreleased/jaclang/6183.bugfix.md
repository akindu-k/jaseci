- **Fix: non-Optional fields stored as `None` no longer quarantine the entire anchor**: When a persisted anchor had `null` stored for a non-Optional field (e.g. `has _conv: list[dict] = []` written as `null` by a past schema migration or write bug), `_deserialize_archetype` raised a `TypeError` that caused the anchor to be dropped and quarantined under the misleading label "class X unresolvable". Because the cascade-quarantine mechanism propagates to every connected `EdgeAnchor`, a single bad field value could quarantine thousands of unrelated documents. The serializer now recovers by restoring the field's declared `default` or `default_factory` value and logging a warning; the anchor loads normally and no quarantine fires. The same recovery applies when a nested object's class has been removed (`_deserialize_value` returns `None` for unresolvable types). Additionally, `SqliteMemory` now distinguishes "class not in registry" (`class X unresolvable`) from "class found but field failed" (`archetype field deserialization failed: X`) so operators can immediately identify the correct recovery path.
