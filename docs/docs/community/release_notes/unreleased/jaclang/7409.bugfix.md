- **Fix: adding or removing an edge on a root no longer serves a stale topology index on other pods**: in multi-pod deployments, type-filtered traversals from a root (`[root ->[?:Type]]`) could keep returning stale results on pods that did not perform the edge write, because the root's cached topology index was never refreshed or invalidated across pods. Edge changes on a root now invalidate its cached index everywhere, so every pod reloads the fresh index.
