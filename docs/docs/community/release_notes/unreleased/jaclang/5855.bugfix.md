- **Fix: topology index updates no longer silently dropped by deferred sync**: when a walker only modified `topology_index_data` (e.g. adding a new node type via `set_topology_index`), the hash-based sync guard skipped the write because topology is excluded from the hash. An explicit `is_topology_dirty` flag on `NodeAnchor` is now set by `set_topology_index` and checked by `SqliteMemory.sync` and `MongoBackend.sync`, ensuring topology-only changes are always persisted.
