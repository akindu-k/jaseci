- **Fix: byllm no longer blocks the event loop during LLM calls**: `Model.model_call_no_stream`, `model_call_with_stream`, and `model_call_http` used the synchronous `openai.OpenAI` client and `requests.post`, pinning the uvicorn event loop for the full LLM round-trip (0.5–30 s) when called from an async walker. Added `BaseLLM.ainvoke()` which offloads the entire call via `asyncio.to_thread`, added async model-call methods (`model_call_no_stream_async`, `model_call_with_stream_async`, `model_call_http_async`) using `AsyncOpenAI`, `litellm.acompletion()`, and `httpx.AsyncClient`, added an `acall_llm` plugin hookimpl for async walkers, and replaced `requests` with `httpx` in the sync HTTP path. Addresses finding 1f from the jac-scale scalability audit (#5874).
